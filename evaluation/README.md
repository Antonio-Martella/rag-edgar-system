# 📊 Evaluation Directory (Benchmarking & Testing)

This directory represents the benchmark for the entire RAG system. It contains validation datasets and automatically generated reports to measure the AI ​​Financial Analyst's accuracy in extracting and reasoning on complex documents (Form 10-K).

All tests in this folder are automatically run by running the script `scripts/run_evaluate_rag.py`.

---

## 📂 Test Data Structure

Tests are logically segmented by company and fiscal year. Currently, the suite evaluates performance on Tesla's (TSLA) 10-K financial statements for the three-year period 2023-2025.

```text
evaluation/
├── eval_tsla_10K_2023/
├── eval_tsla_10K_2024/
└── eval_tsla_10K_2025/
```
**Contents of each folder**:
1. `test_queries_202X.json` **(Ground Truth)**: This file is the reference point. It contains a list of specific questions asked about that year's financial statements and the expected **correct answer** (either manually extracted or verified).
2. `eval_report_202X.json` **(Evaluation Output)**: This is the file dynamically generated at the end of the test. It contains a complete log comparing the expected answer with the answer actually generated by the RAG system.

## ⚖️ Evaluation Methodology (LLM-as-a-Judge)
To ensure an impartial and automated test, the system does not use simple text checks (which would fail if the RAG uses synonyms or different formatting), but employs the **LLM-as-a-Judge** technique.

For each question in the suite:
1. The RAG system generates its answer.
2. An "External Judge" (the same LLM trained with a rigorous evaluation prompt) analyzes the generated answer, comparing it to the ground truth.
3. The Judge verifies whether the key numerical data and the logical sense match.
4. The Judge issues a definitive verdict: **PASS** or **FAIL**.

The results of each question and the total score are then consolidated in the `eval_report_202X.json` file.

## 📈 Test Results and Benchmarks
The following accuracy metrics represent the real-world performance of the RAG system.

**Test Hardware/Software Configuration**:
* **LLM Model**: Mistral-7B-Instruct-v0.2
* **Quantization**: 8-bit (VRAM optimization enabled)
* **Retriever**: FAISS + BAAI Reranker (Top-5 final chunks)
