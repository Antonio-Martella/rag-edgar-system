# 🖥️ Frontend Directory

This directory contains the Edgar RAG Multi-Analyst User Interface (UI). The entire frontend is built using Streamlit, a Python framework ideal for transforming data science scripts and machine learning architectures into interactive and responsive web applications.

The visual architecture was designed following the principle of modularity, separating the chat logic from the side navigation components.

## 📂 File Structure

### 1. `app.py` (Main Entry Point)
This is the heart of the web application and manages the main logic of user-AI interaction.
* **State Management:** Initializes and manages the `st.session_state` to preserve chat history and RAG models loaded into memory across page reloads.
* **Chat Interface:** Renders user messages and Analyst responses. Prepares the *history* of recent messages to provide context to the Large Language Model.
* **Self-Evaluating UI:** Unpacks the dictionaries generated by the backend to feed the "Live Quality Badge" system. Dynamically prints green (✅ Verified) or yellow (⚠️ Incomplete) badges under each response, revealing the internal Judge's reasoning log to the user.

### 2. `components.py` (Modular UI Elements)
This file hosts the secondary visual components to keep `app.py` clean, readable, and scalable.
* **Sidebar (`render_sidebar`):** Contains widgets and forms that allow the user to configure the analysis. It handles the company ticker input (e.g., TSLA, AAPL), fiscal year selection, and document type (e.g., 10-K). When the user presses "Load," it communicates with the backend to perform instant vector data swapping (FAISS) without taxing the GPU.

## 🚀 How to Launch the Interface

Make sure you've first completed document indexing using the backend scripts.
To launch the web application, open the terminal in the main root of the project and run the following command:
```bash
streamlit run frontend/app.py
```
